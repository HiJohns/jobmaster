description: 执行标准的提交和日志记录流程
---
任务已完成。请严格执行以下 'Commit It' 协议：
1. 使用描述性信息 git commit 所有更改。
2. git push 到 origin。
3. 在 docs/blog.md 末尾追加一条 Raw Entry（中文）。
4. 检查 docs/blog.md 中是否有属于【以前日期】的 Raw Entry，如果有，将其合并为当天的 Daily Summary（倒序排列）。在合并时，应遵循以下原则：
   a. **合并同类项**：将针对同一功能、模块或对象的多个零散条目（如多个 UI 修复、多个 Agent 逻辑调整）合并为一个更概括的条目。
   b. **提升概括性**：用更高级、更具总结性的语言描述工作内容，避免罗列琐碎的技术细节。例如，将“修复 A 按钮的颜色”和“调整 B 输入框的边距”合并为“优化了用户设置界面的视觉样式和布局”。
5. 不要为【今天】生成 Summary。
