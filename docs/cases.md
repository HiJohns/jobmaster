## 1. 系统初始化

### 用例 01：超级用户登录

**参与者**：超级用户（系统初始化角色）

**前置条件**：
- 系统已完成部署，处于冷启动状态
- 环境变量 `.env` 中已配置 `INIT_PASSWORD`

**主流程**：
1. 系统启动时检查关联 IAM 系统，确认不存在与 jobmaster 相关的任务用户
2. 系统调用 IAM API 自动创建 jobmaster 超级用户
3. 用户在前端打开页面，系统跳转至 IAM 登录页
4. 用户按约定完成超级管理员登录，获取 JWT Token
5. 用户查看租户列表
6. 用户尝试创建租户 tenant1
7. 用户创建主户管理员 admin@root
8. 用户查看用户列表
9. 用户查看系统日志

**预期结果**：
- 步骤 6 应返回 403，超级用户不允许执行业务操作
- 用户列表中可见主户管理员用户
- 日志中记录超级用户登录及创建主户管理员的操作

## 2. 租户与组织管理

### 用例 02：主户管理员登录

**参与者**：主户管理员（admin@root）

**前置条件**：
- 主户管理员账户已创建

**主流程**：
1. 用户使用主户管理员账户 + 通用密码登录
2. 用户创建租户管理员账户：
   - admin@tenant1
   - admin@tenant2
3. 用户创建工程公司管理员账户（工程公司在体系中视为一种租户）：
   - admin@contractor1
   - admin@contractor2
   - admin@contractor3
4. 用户查看租户列表
5. 用户查看用户列表
6. 用户查看系统日志

**预期结果**：
- 租户列表中可见新建的租户及工程公司
- 用户列表中可见主户管理员及租户管理员，不可见超级用户
- 日志中记录登录及创建租户/管理员的操作

### 用例 03：租户管理员登录

**参与者**：租户管理员（admin@tenant1）

**前置条件**：
- 租户 tenant1 已创建

**主流程**：
1. 用户使用 admin@tenant1 + 通用密码登录
2. 用户查看分公司列表（初始为空）
3. 用户创建分公司 branch1、branch2，同步登记分公司管理员：
   - admin@branch1
   - admin@branch2
4. 用户查看分公司列表
5. 用户查看用户列表
6. 用户使用关键字 "contractor1" 搜索工程公司
7. 用户将 contractor1 分配给 branch1（需提权）
8. 用户查看 contractor1 详情页
9. 用户查看 branch1 详情页
10. 用户查看系统日志

**预期结果**：
- 分公司列表中可见新建的分公司
- 用户列表中可见租户管理员、分公司管理员及工程公司管理员，不可见超级用户和主户管理员
- contractor1 详情页中业务来源包含 branch1
- branch1 详情页中关联工程公司包含 contractor1
- 日志中记录登录、创建分公司/管理员、分配工程公司的操作

## 3. 分公司业务

### 用例 04：分公司管理员登录

**参与者**：分公司管理员（admin@branch1）

**前置条件**：

**主流程**：
1. 用户使用 admin@branch1 + 通用密码登录

8. 用户创建工单 1、2，指派给 contractor1
9. 用户查看工单列表
10. 用户查看系统日志

**预期结果**：
- 用户列表中可见分公司管理员及员工用户
- 关联工程公司列表中可见 contractor1
- 工单列表中可见工单 1、2，状态为 PENDING
- 日志中记录登录、创建员工、创建工单的操作

### 用例 05：分公司员工登录

**参与者**：分公司员工（employee1@branch1）

**前置条件**：
- 分公司 branch1 已创建
- 员工账户 employee1@branch1 已创建

**主流程**：
1. 用户使用 employee1@branch1 + 通用密码登录
2. 用户查看关联的工程公司
3. 用户创建工单 3、4，指派给 contractor1
4. 用户查看系统日志
5. 用户查看工单列表

**子流程：创建工单**
1. 用户点击"创建工单"按钮
2. 填写工单信息：标题、工作内容、设备类型、品牌、所在行政区划（至区/县一级）
3. 上传图片（可选）
4. 提交工单

**子流程：验收工单**
1. 用户查看工单列表，打开 FINISHED 状态的工单 X
2. 检查施工记录（文字留言、照片）
3. 选择"验收通过"或"验收不通过"：
   - **验收不通过**：输入拒单理由（必填）、上传图片（可选），服务端将工单状态转为 DISPATCHED，通知工程公司重新处理
   - **验收通过**：输入评论（可选）、上传图片（可选），服务端将工单状态转为 PENDING_EVALUATION，通知工单所属单位员工进行评估

**子流程：处理预约**
1. 用户查看预约列表，看到工程师发起的预约请求（状态：待确认）
2. 查看预约详情（预约时间、工作内容）
3. 选择操作：
   - **确认预约**：检查时间后点击"确认"，可选输入评论，服务端将预约状态转为已确认，工单状态自动转为 RESERVED
   - **拒绝预约**：输入拒绝原因（必填），点击"拒绝"，预约状态转为已拒绝，工单状态保持 ACCEPTED
4. 已确认的预约可点击"改期"：输入新预约时间及改期原因，等待工程师确认

**预期结果**：
- 关联工程公司列表中可见 contractor1
- 工单 3、4 创建成功，状态为 PENDING
- 工单列表中工单 1~4 均处于等待状态
- 使用 admin@branch1 登录可查看 employee1 创建工单的记录

## 6. 维修执行流程

### 用例 10：维修流程（工程师端）

**参与者**：工程师（engineerX@contractorY）

**前置条件**：
- 工程师已收到微信呼叫通知
- 工单已分配给该工程师

**主流程**：

#### 10.1 工程师接单与预约
1. 工程师使用账户 + 通用密码登录
2. 查看工单列表，确认工单 X 已分配给自己
3. 通过 API 确认接单（acknowledge），设置预约上门时间
4. 服务端将工单状态转为 ACCEPTED
5. 系统通知工单创建人所在分公司（告知工程师信息及预约时间）

#### 10.2 分公司确认预约
1. 分公司员工（employee1@branch1）查看工单列表
2. 打开工单 X，查看工程师及预约时间
3. 点击"确认预约"
4. 服务端将工单状态转为 RESERVED，生成日志

#### 10.3 工程师进场施工
1. 分公司员工打开工单 X，生成进场确认二维码
2. 工程师使用微信端扫描二维码
3. 确认进场位置（自动获取 GPS）
4. 点击"进场"
5. 服务端验证二维码有效性
6. 工单状态转为 WORKING，生成日志
7. 完成施工后点击"结束"
8. 工单状态转为 FINISHED，生成日志

#### 10.4 分公司验收
1. 分公司员工打开工单 X
2. 检查施工记录（文字留言、照片）
3. 点击"验收通过"
4. 服务端将工单状态转为 PENDING_EVALUATION，生成日志
5. 系统自动通知工程师及所在公司

**子流程：工程师发起预约**
1. 工程师查看工单列表，打开 ACCEPTED 状态的工单 X
2. 点击"发起预约"
3. 选择预约日期和时间
4. 输入评论（可选）
5. 提交预约请求
6. 服务端创建预约记录（状态：pending）
7. 通知分公司员工确认

**子流程：工程师改期**
1. 工程师查看已确认的预约
2. 点击"改期"
3. 选择新的日期和时间
4. 输入改期原因
5. 提交改期请求
6. 服务端更新预约时间
7. 通知分公司员工

## 7. 报价与结算

### 用例 11：供应商职员报价

**参与者**：供应商员工（employee@vendor）

**前置条件**：
- 供应商已创建员工账户
- 存在已完成的工单（FINISHED 状态）

**主流程**：
1. 用户使用供应商员工账户登录
2. 查看工单列表，定位已完成工单
3. 进入工单详情，填写报价：
   - 添加报价项目（项目名称、单价、数量）
   - 系统自动计算小计和总价
   - 可动态添加/删除项目行
4. 提交报价单
5. 报价状态转为"待审核"

**查看报价列表**：
1. 进入"报价管理"模块
2. 查看自己提交的报价列表
3. 报价状态：待审核 / 接受 / 拒绝

**预期结果**：
- 报价成功提交，状态为"待审核"
- 报价列表中可见该报价记录

---

### 用例 12：工程公司职员报价

**参与者**：工程公司员工（employee@contractor）

**前置条件**：
- 工程公司已创建员工账户
- 存在已完成的工单（FINISHED 状态）

**主流程**：

#### 12.1 提交报价
1. 用户使用工程公司员工账户登录
2. 查看工单列表，进入已完成工单详情
3. 填写报价并提交（流程同用例 11）

#### 12.2 审核供应商报价
1. 进入"报价管理"模块
2. 切换至"收到的报价"Tab
3. 查看供应商提交的报价列表
4. 进入供应商报价详情：
   - 查看供应商报价明细（项目、单价、数量、小计）
   - 设定加利比例（InputNumber，%）
   - 系统自动计算新总价 = 原总价 × (1 + 加利比例 / 100)
   - 点击"提交"（状态转为"已处理"）或"拒绝"（状态转为"拒绝"）

**预期结果**：
- 自己提交的报价状态流转正常
- 供应商报价审核后状态正确更新
- 加利比例计算准确

## 附录 A：权限说明

### A.1 关联操作必须提权

以下关联组织操作必须在提权（Impersonation）状态下完成：

| 操作类型 | 角色 | 提权后操作 |
|----------|------|------------|
| 给分公司指定工程公司 | 租户管理员 | 将工程公司分配给分公司 |
| 给分公司指定工程公司 | 分公司管理员 | 将工程公司分配给自己所属分公司 |
| 给工程公司指定供应商 | 工程公司管理员 | 将供应商加入工程公司的供应商列表 |

### A.2 关联关系

```
租户 (Tenant)
    │
    ├── 分公司 (Branch)
    │       └── 关联工程公司 (Contractor) ← 需提权操作
    │
    └── 工程公司 (Contractor)
            └── 关联供应商 (Vendor) ← 需提权操作
```

### A.3 权限矩阵

| 角色 | 关联组织操作 | 说明 |
|------|-------------|------|
| 租户管理员 | 给分公司分配工程公司 | 需提权 |
| 分公司管理员 | 给所属分公司分配工程公司 | 需提权 |
| 工程公司管理员 | 添加外部供应商 | 需提权 |
| 普通员工 | 无 | 不可进行关联组织操作 |
