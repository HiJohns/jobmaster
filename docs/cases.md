# 工单匠流程

本文档将描述需要测试的流程。

## 用例01 超级用户登录

前置条件：系统部署完成，冷启动，在.env文件中准备好

- 通用密码（INIT_PASSWORD）

系统检查关联的 IAM 系统，未发现与 jobmaster 相关的任务用户

系统呼叫 IAM API创建jobmaster超级用户，完成启动

用户在前端打开页面，系统跳转到 IAM，按约定完成超级管理员登录，拿到JWT token

用户查看租户列表

用户创建租户tenant1，403（超级用户不允许执行业务操作）

用户创建主户管理员admin@root

用户查看用户列表，可以看到主户管理员用户

用户查看日志，可以看到自己登录、创建主户管理员的记录

## 用例02 主户管理员登录

用户使用主户管理员账户+通用密码登录

用户创建以下账户

- 租户tenant1/tenant2管理员的个人账户(admin@tenant1 / admin@tenant2)
- 工程公司 contractor1 / contractor2 / contractor3同步登录工程公司管理员的个人账户（admin@contractor1 / admin@contractor2 / admin@contractor3），注意工程公司在这个体系中也是一种租户

用户查看租户列表，可以看到新建的租户和新建的工程公司

用户查看用户列表，可以看到主户管理员和租户管理员用户，不应该看到超级用户

用户查看日志，可以看到自己登录、创建各个租户和管理员的记录

## 用例03 租户管理员登录

前置条件：租户tenant1已创建

用户查看用户列表，可以看到租户管理员、分公司管理员和工程公司管理员的用户，看不到超级用户和主户管理员。

用户查看 contractor1详情，可以看到业务来源中有 branch1

## 用例10 维修流程

前置条件：工程师收到微信呼叫

工程师使用engineerX@contractorY+通用密码登录

工程师查看工单列表，可以看到分配给自己的工单 X

工程师通过 API 确认接单（acknowledge），设置预约上门时间

服务端将工单状态转为已接单（ACCEPTED），并通知工单 X 的创建人所在的分公司 branch1（工程师、预约时间）

employee1登录，查看工单列表，打开相应的工单 X ，可以看到工程师和预约时间，点击确认

服务端将工单X状态转为已预约（RESERVED）（生成日志）

employee1登录，打开工单 X，点击生成进场确认二维码，服务端生成二维码返回给前端展示

工程师使用微信端扫描，点击进场，微信端进入工作中状态，呼叫服务端将相应工单状态转为维修中（生成日志）

工程师在微信端点击工作完成离场，

服务器端将相应工单状态转为验收（生成日志）

employee 登录，打开工单 X，点击验收通过，服务器端将工单状态转为完成（生成日志）

系统自动通知工程师和所在公司。

## 用例05 分公司员工操作（补充）

前置条件：employee1@branch1 登录

### 创建工单
1. employee1 点击"创建工单"按钮
2. 填写工单信息：标题、工作内容、设备类型、品牌
3. 上传图片（可选）
4. 提交工单
5. 服务端生成工单号，状态转为 PENDING

### 验收工单
1. employee1 查看工单列表，打开 FINISHED 状态的工单 X
2. 检查施工记录（文字留言、照片）
3. 点击"验收通过"或"验收不通过"
4. 验收不通过时：
   - 输入拒单理由（必填）
   - 上传图片（可选）
   - 点击确认
   - 服务端将工单状态转为 DISPATCHED（回退给工程公司）
   - 通知工程公司重新处理
5. 验收通过时：
   - 输入评论（可选）
   - 上传图片（可选）
   - 点击确认
   - 服务端将工单状态转为 **PENDING_EVALUATION**（待评估）
   - 通知工单所属单位员工进行评估

### 处理预约
1. employee1 查看预约列表
2. 看到工程师发起的预约请求（状态：待确认）
3. 点击预约详情
4. 查看预约时间、工作内容
5. 选择操作：
   - **确认预约**：
     - 检查时间是否合适
     - 点击"确认"
     - 可选输入评论
     - 服务端将预约状态转为已确认
     - 相应工单状态自动转为 RESERVED
   - **拒绝预约**：
     - 输入拒绝原因（必填）
     - 点击"拒绝"
     - 预约状态转为已拒绝
     - 工单状态保持不变（仍为 ACCEPTED）
6. 对于已确认的预约，可点击"改期"：
   - 输入新的预约时间
   - 输入改期原因
   - 等待工程师确认

## 用例10 维修流程（补充）

### 工程师发起预约
1. 工程师查看工单列表，打开 ACCEPTED 状态的工单 X
2. 点击"发起预约"
3. 选择预约日期和时间
4. 输入评论（可选，如："建议上午9点"）
5. 提交预约请求
6. 服务端创建预约记录，状态为 pending
7. 通知分公司员工确认预约

### 工程师改期
1. 工程师查看已确认的预约
2. 点击"改期"
3. 选择新的日期和时间
4. 输入改期原因
5. 提交改期请求
6. 服务端更新预约时间
7. 通知分公司员工

### 工程师扫码施工
1. 工程师在首页点击"扫码"按钮
2. 调用微信扫一扫功能
3. 扫描工单二维码
4. 确认进场位置（自动获取GPS）
5. 点击"进场"
6. 服务端验证二维码有效性
7. 工单状态转为 WORKING
8. 进入施工界面
9. 完成施工后点击"结束"
10. 工单状态转为 FINISHED

## 权限说明：关联组织

### 关联操作必须提权

以下关联组织操作必须在提权（Impersonation）状态下完成：

| 操作类型 | 角色 | 提权后操作 |
|----------|------|------------|
| 给分公司指定工程公司 | 租户管理员 | 将工程公司分配给分公司 |
| 给分公司指定工程公司 | 分公司管理员 | 将工程公司分配给自己所属分公司 |
| 给工程公司指定供应商 | 工程公司管理员 | 将供应商加入工程公司的供应商列表 |

### 关联关系

```
租户 (Tenant)
    │
    ├── 分公司 (Branch)
    │       └── 关联工程公司 (Contractor) ← 需提权操作
    │
    └── 工程公司 (Contractor)
            └── 关联供应商 (Vendor) ← 需提权操作
```

### 权限说明

- **租户管理员**：可给分公司分配工程公司（需提权）
- **分公司管理员**：可给自己所属分公司分配工程公司（需提权）
- **工程公司管理员**：可添加外部供应商到自己公司（需提权）
- **普通员工**：不可进行关联组织操作
