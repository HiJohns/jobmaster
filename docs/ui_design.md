# JobMaster 界面规划文档

本文档描述 JobMaster 系统在 PC 端和微信端的界面设计规划。

---

## 1. 角色与界面对应关系

### 1.1 PC 端界面矩阵

| 角色 | 普通视图 | 提权后视图 | 说明 |
|------|----------|------------|------|
| 超级用户 | 系统配置 | 不支持提权 | 全局系统配置 |
| 主户管理员 | 租户管理 | 不支持提权 | 创建租户、工程公司 |
| 租户管理员 | 租户概览 | 不支持提权 | 查看租户、分公司列表 |
| 分店管理员 | 工单列表/详情 | 分店管理后台 | 创建工单、生成二维码、验收 |
| 分店员工 | 工单列表/详情 | 不支持提权 | 创建工单 |
| 工程公司管理员 | 工单列表/详情 | 工程公司管理后台 | 分配工单、添加供应商 |
| 工程公司员工 | 工单列表/详情 | 不支持提权 | 分配工单 |
| 供应商管理员 | 工单列表/详情 | 供应商管理后台 | 接收工单、分配工程师 |
| 供应商员工 | 工单列表/详情 | 不支持提权 | 分配工单 |

### 1.2 微信端界面矩阵

| 角色 | 界面 | 说明 |
|------|------|------|
| 工程师 | 工单列表 | 查看分配给自己的工单 |
| 工程师 | 工单详情 | 接单、预约时间、进场、离场 |
| 工程师 | 进场扫码 | 扫描工单二维码确认进场 |
| 工程师 | 施工记录 | 提交留言、上传照片 |

## 2. PC 端设计

### 2.1 登录页面

**路由**: `/login`

**功能点**:
- 用户名/邮箱输入框
- 密码输入框
- 记住账号复选框
- 登录按钮
- 忘记密码链接（TODO）

### 2.2 租户选择页面

**路由**: `/select-tenant`

**功能点**:
- 租户列表展示（租户名称、Logo、角色）
- 租户点击选择
- 单租户自动跳转

**触发条件**: 登录后 `GET /api/v1/auth/my-tenants` 返回数组长度 > 1

### 2.3 主布局 (AppLayout)

**包含组件**:
- 顶部栏：租户名称、当前用户、只读模式水印
- 侧边栏：导航菜单
- 面包屑
- 内容区域

### 2.4 超级用户界面

#### 2.4.1 系统配置页

**路由**: `/admin/system`

**功能点**:
- 系统参数配置
- 环境变量管理（只读）
- 系统日志查看

### 2.5 主户管理员界面

#### 2.5.1 租户管理页

**路由**: `/admin/tenants`

**功能点**:
- 租户列表（名称、代码、状态）
- 创建租户按钮 → 模态框
- 租户详情查看
- 编辑租户
- 删除租户（软删除）

#### 2.5.2 工程公司管理页

**路由**: `/admin/contractors`

**功能点**:
- 工程公司列表（名称、代码、关联租户）
- 创建工程公司按钮 → 模态框
- 工程公司详情查看
- 编辑工程公司

#### 2.5.3 用户管理页

**路由**: `/admin/users`

**功能点**:
- 用户列表（姓名、邮箱、角色、租户）

### 2.6 租户管理员界面

**功能点**:

- 分公司列表
- 关联工程公司列表

#### 2.6.2 分公司管理页

**路由**: `/tenant/branches`

**功能点**:
- 分公司列表
- 创建分公司 → 模态框

### 2.7 分店管理员界面

#### 2.7.1 普通视图：工单列表

**路由**: `/orders`

**功能点**:
- 周日历选择器
- 状态 Tabs：待服务、服务中、待修正、已完成
- 搜索框（模糊搜索工单号、地址）
- 工单卡片列表
- 创建工单按钮

#### 2.7.2 普通视图：工单详情

**路由**: `/orders/:id`

**功能点**:
- 工单基本信息（单号、状态、创建时间）
- 设备信息（类型、品牌、位置）

- 时间轴（各状态变更时间）
- 费用信息（labor_fee, material_fee, other_fee）
- **生成进场二维码按钮**（状态 = RESERVED/ARRIVED）
- **验收通过按钮**（状态 = FINISHED）
- **验收不通过按钮**（状态 = FINISHED）

#### 2.7.3 提权视图：分店管理后台

**路由**: `/store-admin`

**功能点**:
- 员工列表
- 创建员工 → 模态框
- 关联工程公司列表
- 工单统计面板

#### 2.7.4 创建工单模态框

**功能点**:
- 设备类型选择
- 品牌选择
- 故障分类选择
- 故障描述（文本）

## 3. 微信端设计

### 3.1 微信端架构

- 微信小程序或 H5（使用微信 JS-SDK）
- 微信授权登录（静默获取 openid）
- 轻量化移动端 UI

### 3.2 微信登录页

**路由**: `/wechat/login`

**功能点**:
- 微信授权按钮
- 自动登录（静默）
- 绑定已有账户（首次登录）

### 3.3 工程师首页

**路由**: `/wechat/orders`

**功能点**:
- 工单列表（分配给我的）
- 状态徽章（已预约/已到场/施工中）
- 工单数量统计

### 3.4 工程师工单详情

**路由**: `/wechat/orders/:id`

**功能点**:
- 工单基本信息
- 设备信息
- 地理位置（地图展示）
- **接单按钮**（状态 = DISPATCHED）
- **预约时间选择器**（状态 = DISPATCHED）
- **扫码进场按钮**（状态 = RESERVED，调用微信扫一扫）
- **离场确认按钮**（状态 = WORKING）
- 状态时间轴

### 3.5 微信扫码进场页

**路由**: `/wechat/arrive/:qrcode`

**功能点**:
- 扫码结果展示
- 确认进场按钮
- GPS 位置获取（微信 JSSDK）
- 拍照上传（到场照片）

### 3.6 工程师施工记录页

**路由**: `/wechat/orders/:id/record`

**功能点**:
- 文字留言输入框
- 照片上传（多张，最多 9 张）
- 照片预览与删除
- 提交按钮

### 3.7 微信端公共组件

- 底部导航栏：首页、我的
- 导航栏：返回按钮 + 标题
- 加载状态指示
- 错误提示 Toast

## 4. 提权机制说明

### 4.1 提权流程

```
1. 普通用户点击"进入管理后台"
2. 调用 POST /api/v1/auth/impersonate
3. 服务端返回新的 JWT token（包含 impersonator_id）
4. 前端切换 token，进入提权视图
5. 点击"退出管理界面"
6. 调用 POST /api/v1/auth/exit-impersonate
7. 服务端返回原普通 token
```

### 4.2 提权后界面变化

| 组件 | 普通视图 | 提权视图 |
|------|----------|----------|
| 顶部栏 | 无变化 | 显示"提权中"标识 |
| 侧边栏 | 业务菜单 | 管理菜单 |
| 工单操作 | 业务操作 | 管理操作 |
| 只读模式 | 不适用 | 提权后不受 is_impersonated 影响 |

## 5. 界面状态管理

### 5.1 只读模式

当 `is_impersonated = true` 时：
- 所有输入框禁用
- 所有操作按钮隐藏
- 仅展示数据

### 5.2 状态联动

| 工单状态 | 分店操作 | 工程公司操作 | 供应商操作 | 工程师操作 |
|----------|----------|--------------|------------|------------|
| PENDING | 创建工单 | 分配工单 | - | - |
| DISPATCHED | 查看 | 分配工程师/供应商 | 分配工程师 | 接单 |
| RESERVED | 生成二维码 | 查看 | 查看 | 扫码进场 |
| ARRIVED | 查看 | 查看 | 查看 | 开始施工 |
| WORKING | 查看 | 查看 | 查看 | 提交施工记录 |
| FINISHED | 验收 | 查看 | 查看 | - |
| OBSERVING | 验收 | 查看 | 查看 | - |
| CLOSED | 查看 | 查看 | 查看 | - |
