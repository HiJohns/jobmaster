# JobMaster 界面规划文档

本文档描述 JobMaster 系统在 PC 端和微信端的界面设计规划。

---

## 0. 设计目标与原则

### 0.1 产品本质

> **JobMaster = 多角色协同的工单流转系统（B2B + 运维调度）**

### 0.2 核心体验目标

1. **降低复杂度（多角色）**
2. **强化状态感（工单流转）**
3. **提升操作效率（高频任务）**

### 0.3 设计原则

#### 原则1：角色无感化

> **不要让用户感知"角色"，而是感知"我能做什么"**

- 菜单按"任务"而不是"角色"组织
- 同一页面不同权限 → 控制按钮，而不是换页面
- **一套 UI + 权限控制 > 多套 UI**

#### 原则2：工单为核心对象

所有设计围绕：
- 工单状态
- 工单流转
- 工单责任人

#### 原则3：状态驱动 UI

> **按钮 = 状态机的可视化**

不允许出现无效按钮（禁止灰显一堆）

#### 原则4：操作优先级

每个页面只能有 **1 个主操作（Primary Action）**

| 页面 | 主操作 |
|------|--------|
| 工单列表 | 创建工单 |
| 工单详情 | 当前状态操作（如验收） |

## 3. PC 端设计

### 3.1 主布局 (AppLayout)

**包含组件**:
- 顶部栏：租户名称、当前用户、只读模式水印
- 侧边栏：导航菜单
- 面包屑
- 内容区域

#### 侧边栏结构（统一）

**一级菜单**:
```
工作台（Dashboard）
工单管理（Orders）
组织管理（Org）
系统管理（Admin）
```

**按角色动态显示**，而不是不同系统。

### 3.2 Dashboard（工作台）

**必须实现**：不同角色显示不同卡片。

#### 分公司管理员

| 卡片 | 内容 |
|------|------|
| 今日工单数 | 数字 + 趋势 |
| 待处理 | PENDING/ACCEPTED 状态数量 |
| 进行中 | RESERVED/ARRIVED/WORKING 状态数量 |
| 异常工单 | 超时或验收不通过数量 |

#### 工程公司

| 卡片 | 内容 |
|------|------|
| 待分配 | DISPATCHED 状态数量 |
| 工程师负载 | 各工程师工单数图表 |

### 3.3 登录页面

**路由**: `/login`

**功能点**:
- 用户名/邮箱输入框
- 密码输入框
- 记住账号复选框
- 登录按钮
- 忘记密码链接（TODO）

### 3.4 租户选择页面

**路由**: `/select-tenant`

**功能点**:
- 租户列表展示（租户名称、Logo、角色）
- 租户点击选择
- 单租户自动跳转

**触发条件**: 登录后 `GET /api/v1/auth/my-tenants` 返回数组长度 > 1

### 3.5 工单列表页

**路由**: `/orders`

**布局**:
- 顶部：周日历选择器 + 状态筛选 Tabs
- 中间：工单卡片列表
- 底部：分页

#### 工单卡片（核心组件）

**卡片信息**:
- 工单号
- 地址
- 状态（大标签，颜色编码）
- 当前负责人
- 时间（预约/创建）
- 紧急标记（红色角标）

**设计规则**:
- 卡片 > 表格（更直观）
- 表格仅用于管理后台（用户/租户）

### 3.6 工单详情页

**路由**: `/orders/:id`

**功能点**:
- 工单基本信息（单号、状态、创建时间）
- 设备信息（类型、品牌、位置）
- 地理位置（地址、坐标）
- 时间轴（各状态变更时间）
- 费用信息（labor_fee, material_fee, other_fee）

#### 状态驱动按钮

| 状态 | 显示按钮 |
|------|----------|
| DISPATCHED | [分配工程师] [分配供应商] |
| ACCEPTED | [确认预约时间] |
| RESERVED | [生成二维码] |
| FINISHED | [验收通过] [验收不通过] |
| OBSERVING | [验收通过] [验收不通过] |

### 3.7 提权视图设计（高风险点）

#### 全局警告条（类似 Stripe）

```
┌─────────────────────────────────────────────────────────────────┐
│ ⚠️ 你正在以【管理员身份】操作                    [退出管理后台] │
└─────────────────────────────────────────────────────────────────┘
```

#### UI 变化（必须明显）

| 组件 | 普通视图 | 提权视图 |
|------|----------|----------|
| 顶部栏 | 无变化 | **红色/紫色背景** + "提权中"标识 |
| 页面 | 正常 | 轻微灰化背景 |
| 按钮 | 正常 | 增加 "管理操作" 标签 |
| 侧边栏 | 业务菜单 | 管理菜单 |

#### 操作确认（二次确认）

以下操作必须弹出确认框：
- 删除用户
- 分配关系
- 验收不通过

### 3.8 创建工单模态框

**功能点**:
- 设备类型选择
- 品牌选择
- 故障分类选择
- 故障描述（文本）
- 照片上传（多张）
- 紧急程度开关

## 4. 微信端设计

### 4.1 设计目标

> **3秒内完成操作**

### 4.2 微信端架构

- 微信小程序或 H5（使用微信 JS-SDK）
- 微信授权登录（静默获取 openid）
- 轻量化移动端 UI

### 4.3 工程师首页

**路由**: `/wechat/orders`

**布局**:
- 顶部：今日工单数 + 状态统计（横向滚动）
- 中间：工单卡片（大卡片）
- 底部：Tab 导航（首页 / 我的）

### 4.4 工程师工单详情

**路由**: `/wechat/orders/:id`

**设计：Step Flow（流程化）**

```
接单 → 预约 → 到场 → 施工 → 离场
```

每一步是一个**大按钮**（高亮、巨大、易点击）。

**功能点**:
- 工单基本信息
- 设备信息
- 地理位置（地图展示）
- **接单按钮**（状态 = DISPATCHED）- 主操作，唯一大按钮
- **预约时间选择器**（状态 = ACCEPTED）
- **扫码进场按钮**（状态 = RESERVED，调用微信扫一扫）
- **离场确认按钮**（状态 = WORKING）
- 状态时间轴

### 4.5 扫码体验优化

**优化点**:
- 扫码后自动跳转（不要确认页过多）
- 自动获取 GPS
- 自动打开相机
- 减少点击次数

### 4.6 施工记录页

**路由**: `/wechat/orders/:id/record`

**功能点**:
- 文字留言输入框
- 照片上传（多张，最多 9 张）
- 照片预览与删除
- 提交按钮

**上传策略**:
- 先上传后提交（异步模式）
- 工程师拍一张传一张
- 点击提交时只提交已生成的 URL 数组
- 防止地下室信号差导致表单重填

### 4.7 分公司员工端（微信）

#### 4.7.1 工单列表页

**路由**: `/wechat/orders`

**功能描述**:
- 显示本分公司发起且未关闭的所有工单
- 工单卡片内容：标题、创建时间、创建人、当前状态
- 点击工单卡片 => 跳转到工单详情

**操作权限**:
- 创建工单（跳转创建表单）
- 查看工单详情

#### 4.7.2 工单详情页

**路由**: `/wechat/orders/:id`

**状态操作**:

| 工单状态 | 可执行操作 |
|----------|----------|
| FINISHED | **验收通过** - 工单转为 OBSERVING 或 CLOSED <br> **验收不通过** - 工单转为 REJECTED，回转给工程公司 |
| REJECTED | 查看拒单理由 |
| DISPATCHED, ACCEPTED, RESERVED, WORKING | 显示二维码（扫码进场/离场） |

**验收操作细节**:
- 验收通过：输入评论（可选），上传图片（可选）
- 验收不通过：必须输入评论说明原因，工单状态转为拒单

#### 4.7.3 预约页

**路由**: `/wechat/reservations`

**预约列表**:
- 显示字段：预约时间、工作内容（相关工单标题）、预约者、状态
- 状态：待确认、协商中、已确认、已过期
- 点击预约卡片 => 跳转到预约详情

**预约详情路由**: `/wechat/reservations/:id`

**状态操作**:

| 预约状态 | 可执行操作 |
|----------|----------|
| 待确认 | **确认** - 预约状态转为已确认，相应工单转入 RESERVED <br> **拒绝** - 拒绝时可输入评论，预约状态转为已拒绝 |
| 已确认 | **改期** - 弹出预约对话框（指定时间并加评论） |
| 已过期 | 仅查看，不可操作 |

### 4.8 工程公司员工端（微信）

**路由**: `/wechat/orders`

**工单列表范围**: 显示本工程公司相关的所有工单（流转、分配、已预约、施工中、验收、拒收、拒单）

**可执行操作**:

| 工单状态 | 可执行操作 |
|----------|----------|
| DISPATCHED | **转发** - 转发表单（选择供应商/工程公司） <br> **分配** - 分配工程师 |
| ACCEPTED | **转发** - 转发表单 <br> **分配** - 分配工程师 |
| REJECTED | **分配** - 重新分配工程师 <br> **拒单** - 拒单对话框输入拒单理由，回转给分公司 |
| RESERVE (逾期) | **转发** - 转发表单 <br> **分配** - 分配工程师 |
| RESERVED | 查看预约信息 |
| FINISHED | **验收** - 查看验收状态 |

### 4.9 供应商员工端（微信）

**路由**: `/wechat/orders`

**工单列表范围**: 显示本供应商相关的所有工单（流转、分配、已预约、施工中、验收、拒收）

**可执行操作**:

| 工单状态 | 可执行操作 |
|----------|----------|
| DISPATCHED | **转发** - 转发表单（选择其他供应商/工程公司） <br> **分配** - 分配工程师 |
| ACCEPTED | **转发** - 转发表单 <br> **分配** - 分配工程师 |
| REJECTED | **分配** - 重新分配工程师 <br> **拒单** - 拒单对话框输入拒单理由，回转给分公司 |
| RESERVE (逾期) | **转发** - 转发表单 <br> **分配** - 分配工程师 |
| RESERVED | 查看预约信息 |
| FINISHED | **验收** - 查看验收状态 |

### 4.10 预约日志（工程师端）

**路由**: `/wechat/orders/:id`

**预约日志显示位置**: 工单详情页底部

**日志内容**:
- 预约提议时间
- 预约状态变更（提议、拒绝、确认、改期）
- 变更时间
- 变更人
- 评论/原因

**状态操作**:

| 工单状态 | 可执行操作 |
|----------|----------|
| ACCEPTED | **发起预约** - 对话框指定时间，并加评论 |
| RESERVED | **改期** - 弹出预约对话框（指定时间并加评论） |

### 4.11 扫码施工流程（工程师）

**扫码入口**: 首页二维码扫描

**流程**:
1. 工程师点击首页"扫码"按钮
2. 调用微信 JS-SDK 扫描二维码
3. 扫码成功后跳转到施工界面
4. 工单状态自动进入 WORKING

**施工界面路由**: `/wechat/orders/:id/work`

**施工界面功能**:
- 显示工单基本信息
- **结束施工**按钮 - 点击后工单进入 FINISHED 状态
- 施工记录入口（跳转到施工记录页）

## 5. 提权机制说明

### 5.1 提权流程

```
1. 普通用户点击"进入管理后台"
2. 调用 POST /api/v1/auth/impersonate { tenant_id: "uuid" }
3. 服务端返回新的 JWT token（包含 impersonator_id）
4. 前端切换 token，进入提权视图
5. 点击"退出管理界面"
6. 调用 POST /api/v1/auth/exit-impersonate
7. 服务端返回原普通 token
```

### 5.2 提权视图变化

| 组件 | 普通视图 | 提权视图 |
|------|----------|----------|
| 顶部栏 | 无变化 | 深色背景 + "提权中"标识 |
| 侧边栏 | 业务菜单 | 管理菜单 |
| 工单操作 | 业务操作 | 管理操作 |
| 只读模式 | 不适用 | 提权后不受 is_impersonated 影响 |

## 6. 界面状态管理

### 6.1 只读模式

当 `is_impersonated = true` 时：
- 所有输入框禁用
- 所有操作按钮隐藏
- 仅展示数据

### 6.2 状态联动

| 工单状态 | 分公司操作 | 工程公司操作 | 供应商操作 | 工程师操作 |
|----------|----------|--------------|------------|------------|
| PENDING | 创建工单 | 分配工单 | - | - |
| DISPATCHED | 查看 | 分配工程师/供应商 | 分配工程师 | 响应接单(acknowledge) |
| ACCEPTED | 确认预约时间 | 查看 | 查看 | 设置预约时间 |
| RESERVED | 生成二维码 | 查看 | 查看 | 扫码进场 |
| ARRIVED | 查看 | 查看 | 查看 | 开始施工 |
| WORKING | 查看 | 查看 | 查看 | 提交施工记录 |
| FINISHED | 验收 | 查看 | 查看 | - |
| OBSERVING | 验收 | 查看 | 查看 | - |
| CLOSED | 查看 | 查看 | 查看 | - |
