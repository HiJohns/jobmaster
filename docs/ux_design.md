# 一、整体设计目标（核心定位）

### 🎯 产品本质

JobMaster ≠ 普通后台系统，而是一个：

> **多角色协同的工单流转系统（B2B + 运维调度）**

所以 UI 目标必须是：

### ✅ 3个核心体验目标

1. **降低复杂度（多角色）**
2. **强化状态感（工单流转）**
3. **提升操作效率（高频任务）**

---

# 二、全局 UI/UX 设计原则（最重要）

## 1️⃣ “角色无感化”原则（核心）

当前问题：
👉 角色太多（8种），UI很容易割裂

### ✅ 设计原则

> **不要让用户感知“角色”，而是感知“我能做什么”**

### 👉 设计方式

* 菜单按“任务”而不是“角色”组织
* 同一页面不同权限 → 控制按钮，而不是换页面

#### ❌ 当前问题

* 分公司管理员 vs 员工 → 两套逻辑

#### ✅ 优化建议

统一为：

* `/orders` 永远一样
* 权限差异：

  * 是否显示按钮（创建 / 验收 / 二维码）
  * 是否可编辑

👉 **一套UI + 权限控制 > 多套UI**

---

## 2️⃣ 工单 = 产品核心对象（Single Source of Truth）

你系统的本质只有一个核心：

> **工单（Order）**

### ✅ 所有设计围绕：

* 工单状态
* 工单流转
* 工单责任人

---

### 👉 UI强化方式（非常关键）

#### 1. 状态视觉化（必须做）

用颜色 + 标签：

| 状态         | 颜色 |
| ---------- | -- |
| PENDING    | 灰  |
| DISPATCHED | 蓝  |
| RESERVED   | 橙  |
| WORKING    | 紫  |
| FINISHED   | 绿  |
| CLOSED     | 黑  |

👉 在：

* 列表
* 详情
* 时间轴

全部统一

---

#### 2. 工单卡片 > 表格（强烈建议）

你现在写的是“列表”，但没定义形式。

👉 建议：

### PC端：

* **卡片 + 简表混合**

卡片信息：

* 工单号
* 地址
* 状态（大标签）
* 当前负责人
* 时间（预约/创建）

👉 表格只用于：

* 管理后台（用户/租户）

---

## 3️⃣ 提权（Impersonation）必须强提示（高风险点）

这是你系统**最容易出事故的地方**

### ❗ 当前问题：

文档只说“深色背景 + 提权中”

👉 不够！！！

---

### ✅ 必须做到：

#### 1. 全局警告条（类似 Stripe）

```
你正在以【管理员身份】操作
[退出]
```

#### 2. UI变化（必须明显）

* 顶栏：红色 or 紫色
* 页面：轻微灰化
* 按钮：增加 “管理操作” 标签

#### 3. 操作确认（二次确认）

例如：

* 删除用户
* 分配关系

👉 必须弹确认

---

## 4️⃣ 状态驱动 UI（State-driven UI）

你已经有一张状态表（很好）

但 UI 没设计出来 👇

---

### ✅ 核心原则：

> **按钮 = 状态机的可视化**

---

### 👉 示例（工单详情页）

不同状态显示不同按钮：

#### DISPATCHED

* [分配工程师]
* [分配供应商]

#### RESERVED

* [生成二维码]

#### FINISHED

* [验收通过]
* [验收不通过]

👉 不允许出现无效按钮（禁止灰一堆）

---

## 5️⃣ 操作优先级（减少认知负担）

### ❗ 当前问题

功能很多，但没有“主次”

---

### ✅ 设计规则：

#### 每个页面：

👉 **只能有 1 个主操作（Primary Action）**

例如：

| 页面   | 主操作         |
| ---- | ----------- |
| 工单列表 | 创建工单        |
| 工单详情 | 当前状态操作（如验收） |

---

### 👉 UI规则

* 主按钮：高亮（蓝色）
* 次按钮：outline
* 危险操作：红色

---

# 四、微信端 UX 优化（非常关键）

移动端不是“PC缩小版”

---

## 1️⃣ 核心目标

> **3秒内完成操作**

---

## 2️⃣ 首页优化（你现在太弱）

当前：

* 只是列表 ❌

---

### ✅ 建议：

#### 顶部：

* 今日工单数
* 状态统计（横向滚动）

#### 中间：

* 工单卡片（大卡片）

#### 底部：

* Tab：首页 / 我的

---

## 3️⃣ 工单详情（移动端重点）

必须做成：

### 👉 Step Flow（流程化）

```
接单 → 预约 → 到场 → 施工 → 离场
```

👉 每一步是一个大按钮

---

## 4️⃣ 扫码体验（关键）

### 优化点：

* 扫码后自动跳转（不要确认页过多）
* 自动获取 GPS
* 自动打开相机

👉 减少点击
